Sol Cleaners — User Guide

Everything you need to know about scheduling pickups, tracking orders, and managing your dry cleaning from any device.

Overview

Sol Cleaners is a professional dry cleaning & laundry service app. Customers can schedule pickups, track orders, and manage their account on iPhone, Android, and any web browser. Admins can manage orders, optimize routes, and view business analytics — all from the same app.

📱
Native Mobile
Download via Expo Go on iOS or Android for the full native experience including biometric login.
🌐
Web / PWA
Open the URL in any browser. Install as a Progressive Web App for an app-like experience on desktop & mobile.
☁️
Always Synced
Orders sync automatically to the cloud. Works offline and catches up the moment you reconnect.
🔐
Secure
Supabase Auth with email/password, Face ID, and fingerprint. All data encrypted in transit and at rest.

Accessing the App

Sol Cleaners can be used on multiple platforms — choose whichever suits you best.

On a Mobile Phone (iOS or Android)

  1. Install Expo Go Download Expo Go from the App Store (iPhone) or Google Play Store (Android). It's free.
  2. Open the link Tap the Sol Cleaners link shared with you, or scan the QR code. The app loads instantly.

In a Web Browser (Desktop or Mobile)

Simply visit the hosted URL (e.g., https://app.solcleaners.com) in any modern browser — Chrome, Safari, Firefox, or Edge. No installation required.

💡
Install as Home Screen App

In Chrome on Android: tap the three-dot menu → Add to Home Screen. On iPhone Safari: tap Share → Add to Home Screen. The app icon appears on your home screen and opens full-screen, just like a native app.

Store Phone Number

You can always reach Sol Cleaners directly: (781) 784-3937. The "Call Us" button inside the app dials this number instantly.

Customer Guide

Create an Account

Register

You need an account to place orders. Registration takes about 60 seconds.

  1. Open the app and tap "New here? Create account" Found below the Sign In button on the welcome screen.
  2. Fill in your details Enter your full name, phone number, email address, street address, city, and zip code.
  3. Choose a password Minimum 6 characters. You'll use this to sign in on any device.
  4. Tap "Create Account" Your account is created and you're signed in automatically. Your name will appear on all your screens.
✅
Already a Sol Cleaners POS customer?

If you already have an account in the in-store system, register with the same phone number. Your history will be linked automatically.

Log In

  1. Enter your email and password The same credentials you used when registering.
  2. Tap "Sign In" You're taken to your home screen. Your name appears in the top greeting.

Forgot Password

  1. Tap "Forgot password?" Below the sign-in button.
  2. Enter your email addressThe one you registered with.
  3. Tap "Send Reset Link"Check your inbox for an email from Sol Cleaners. Click the link in the email.
  4. Set a new passwordThe link opens the app and prompts you to create a new password.
⚠️
Check your spam folder

Password reset emails sometimes end up in spam or junk mail. If you don't see it within 2 minutes, check there before requesting another.

Auto-Login

Once you sign in, the app remembers your session. You won't need to enter your password again unless you manually sign out or are inactive for more than 60 days. This works across app restarts and browser refreshes.

Biometric Login (Face ID / Fingerprint)

On supported mobile devices, you can enable biometric authentication so you never type a password again.

📱
Biometric is only available on native mobile (iOS & Android)

This feature is not available in the web browser version. Web users sign in with email and password.

  1. Sign in once with email & passwordBiometric requires at least one successful password login to save credentials.
  2. Go to My Account → Security SettingsScroll down on your profile screen to the Security Settings card.
  3. Tap "Enable Face ID" or "Enable Fingerprint"The app prompts you to authenticate with your device biometric to confirm.
  4. Done!On your next visit, tap the biometric button on the login screen to sign in instantly.

To disable biometric, tap the toggle again in Security Settings. Password login is always available as a fallback.

Deployment

PWA Overview

Sol Cleaners is a Progressive Web App (PWA). When hosted at a public HTTPS URL, browsers treat it like a native app:

  • Installable to the home screen on iOS (Safari) and Android (Chrome) — no App Store needed
  • Runs full-screen with no browser chrome, just like a native app
  • Branded with the Sol Cleaners name, navy theme colour, and favicon
  • Loads fast — assets are cached after first visit
  • Works offline for viewing cached data; syncs when back online
🚀
No App Store approval needed

Publishing to a web host is instant — no Apple or Google review process. Updates go live immediately for all users.

Deployment

Deploy to Hostinger

Hostinger is a shared hosting provider that supports static file hosting. The Expo web build produces a static site that uploads directly — no server configuration required.

Step 1 — Build the Web Bundle

Run the following command in the sol-native/ project folder:

npx expo export --platform web

This creates a dist/ folder containing the complete static web app — HTML, JavaScript, CSS, and asset files. The build typically takes 1–3 minutes.

💡
What's in dist/?

The dist/ folder contains index.html, bundled JavaScript chunks, fonts, images, and a metadata.json. Every file in this folder must be uploaded to Hostinger.

Step 2 — Upload to Hostinger

  1. Log in to your Hostinger account Go to hpanel.hostinger.com and sign in.
  2. Open File Manager From your hosting dashboard, go to Hosting → Manage → File Manager.
  3. Navigate to public_html This is the root folder for your website. Delete any existing placeholder files (like the default index.html).
  4. Upload the contents of dist/ Click Upload Files and select all files and folders inside your local dist/ folder. Do not upload the dist/ folder itself — upload its contents.
  5. Verify index.html is in public_html root After upload, public_html/index.html should exist. That's the entry point for the app.
📁
Alternative: Upload via FTP

You can also use an FTP client like FileZilla. Get your FTP credentials from Hostinger → Hosting → Manage → FTP Accounts. Connect and drag the contents of dist/ to the /public_html/ directory.

Step 3 — Domain & HTTPS Setup

  1. Point your domain to Hostinger If you have a custom domain (e.g., app.solcleaners.com), go to Hostinger → Domains → DNS Zone and set an A record pointing to your Hostinger server IP shown in the dashboard.
  2. Enable SSL (Free) Go to Hosting → Manage → SSL. Click Install to enable the free Let's Encrypt certificate. This gives you HTTPS — required for PWA install prompts.
  3. Force HTTPS In Hosting → Manage → .htaccess, add the following redirect to ensure all visitors use the secure URL: 
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# SPA fallback — send all routes to index.html
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ /index.html [L]
⚠️
SPA fallback is required

The last three lines of the .htaccess above are critical. Without them, refreshing the page or navigating directly to any URL other than / will return a 404 error from the server.

Updating the App

When you make code changes and want to publish an update:

  1. Run the build againnpx expo export --platform web
  2. Delete the old files in public_htmlIn Hostinger File Manager, select all files and delete them.
  3. Upload the new dist/ contentsSame process as before. No server restart needed — the update goes live instantly.

Troubleshooting

App won't load in browser

  • Clear browser cache and reload with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac).
  • Make sure you are visiting the https:// URL, not http://.
  • Check that index.html exists at the root of public_html.
  • Verify the .htaccess SPA fallback rules are in place.

Page returns 404 on refresh

The .htaccess SPA fallback is missing or incorrect. Paste the exact rules from the Hostinger deployment section into your .htaccess file in public_html.

Login fails ("Invalid credentials")

  • Check that Caps Lock is not on.
  • Use the Forgot Password flow to reset your password.
  • Ensure you are using the email address you registered with — not a phone number or username.

Orders not syncing / showing "Offline"

  • Check your internet connection.
  • Pull down to refresh on the orders screen.
  • Orders created offline will sync automatically when you reconnect. A "Syncing…" indicator appears briefly.

Biometric not appearing on login screen

  • Ensure you have enabled it in My Account → Security Settings.
  • Biometric is only available on native iOS/Android — not the web version.
  • Your device must have biometric hardware configured in device settings (Face ID enrolled on iPhone, fingerprint set up on Android).

PWA install banner not showing

  • The site must be served over HTTPS. Install the SSL certificate in Hostinger.
  • On iOS, use Safari — PWA install is not available in Chrome on iOS.
  • On Android, the banner may take a few visits to appear. Use Chrome's three-dot menu → Add to Home Screen manually.

Build fails with "Cannot find module"

npm install --legacy-peer-deps

Port already in use

npx expo start --web --port 8083

FAQ

Is the web version the same as the mobile app?

Yes — it's the exact same codebase. Every screen, feature, and data flows to the same Supabase cloud backend. The only differences are that biometric login (Face ID/Fingerprint) requires native iOS/Android, and local SQLite storage uses in-memory storage on web (your cloud data is always available).

Can multiple people use the admin portal at the same time?

Yes. Each admin role logs in separately. Order status changes made by one admin are synced to the cloud and visible to others after refreshing.

What happens if I place an order offline?

The order is saved locally on your device. As soon as your device reconnects to the internet, the order is automatically uploaded to the cloud and appears in the admin dashboard.

How do I change the store phone number?

The phone number is defined in App.js. Search for (781) 784-3937 and update the value, then rebuild and redeploy.

How do I add more pickup time slots?

In App.js, update the WEEKDAY_TIME_SLOTS, WEEKEND_TIME_SLOTS, or SUNDAY_TIME_SLOTS arrays near the top of the file.

Can I add more item types?

Yes. Edit the ITEM_TYPES array in App.js. Add a new object with a unique key, a label, and an icon emoji. No other changes are needed.

How do I update the app on Hostinger after a code change?

  1. Run npx expo export --platform web
  2. Delete all files in public_html/ on Hostinger
  3. Upload the new contents of dist/

Is customer data secure?

All data is transmitted over HTTPS and stored in Supabase's encrypted PostgreSQL database. Passwords are hashed by Supabase Auth and never stored in plain text. Biometric credentials are stored in the device's secure enclave and are never sent to the server.
